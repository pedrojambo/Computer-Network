--INTRODUÇÃO

As redes de computadores desempenham um papel fundamental na conectividade global, proporcionando a infraestrutura que sustenta a comunicação instantânea e a transferência eficiente de dados em escala mundial. Em meio a essa complexidade de interconexões, o roteamento emerge como um elemento vital para otimizar o fluxo de informações. 

Este artigo se propõe a realizar uma análise abrangente do conteúdo de roteamento em redes de computadores, explorando a eficácia de diferentes algoritmos por meio da aplicação do software Sinalgo. Em meio a essa análise, conceitos básicos de roteamento de pacotes, bem como os diferentes tipos de algoritmos utilizados atualmente para tal, serão abordados para a compreensão de uma conclusão embasada.

--EXPERIMENTO

Nesse artigo, um experimento será conduzido para avaliar, através do software Sinalgo, o resultado de simulações de execuções de três algoritmos de roteamento diferentes (disponibilizados pelo software). Ao executarmos uma simulação, é possível definir como será o mapeamento dos roteadores na área simulada, permitindo a configuração do posicionamento dos roteadores, número de roteadores posicionados, número do roteador de origem (o roteador que está enviando um pacote), número do roteador de destino (o roteador que é o destino do pacote enviado). Ao término de cada round, o software imprime 3 valores no terminal, respectivamente: o custo de rede da execução, o tempo de execução do algoritmo selecionado e o tempo total.

Para cada algoritmo, serão executadas 30 rodadas da simulação, utilizando o mesmo mapa de roteadores para todas as simulações, com o intuito de comparar a eficiência dos três algoritmos. Ademais, para todas as execuções, o roteador de origem será o de número 43, enquanto que o de destino será 119 para o primeiro comparativo e 44 para o segundo. Ao final das execuções, teremos dois conjuntos de três resultados da simulação (um conjunto para cada um dos dois roteador de destino), cada resultado contendo 30 linhas de resultados de rounds de execução. Por fim, calcularemos o valor do Timeoutinterval para cada round, através do cálculo do EstimatedRTT e DevRTT com os valores de alfa = 0,125 e beta igual a 0,25, para que seja possível visualizar devidamente a comparação proposta.
